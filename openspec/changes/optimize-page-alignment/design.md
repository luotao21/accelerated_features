## Context

单一 Homography 无法正确处理书本折叠时的透视变形。书本展开角度小于 180° 时，左右两页形成两个不同的平面，需要分别建模。

## Goals / Non-Goals

**Goals:**

- 支持书本在 90°-180° 展开角度下的准确热点对齐
- 保持实时性能（>5 FPS）
- 最小化对现有代码的改动

**Non-Goals:**

- 不处理极端展开角度（<90°）
- 不建模页面的实际曲率（简化为两个平面）

## Decisions

### 决策 1：使用双 Homography 而非 TPS

**原因**：TPS 需要更多的控制点和更复杂的计算，而双 Homography 足以处理书页折叠这一特定场景。

### 决策 2：基于 x 坐标分区

**原因**：书脊通常在参考图像的中线附近，可以简单地使用 x 坐标阈值进行分区。未来可以添加自动书脊检测。

### 决策 3：热点继承分区逻辑

**原因**：每个动物热点的 SVG 边界框可以确定其属于左页还是右页。

## Risks / Trade-offs

- **风险**：脊线位置估计不准确 → 缓解：允许用户配置脊线位置
- **风险**：左右页特征点不均匀 → 缓解：各自要求最小匹配数

## Open Questions

1. 是否需要自动检测书脊位置，还是使用固定中线？
2. 当一侧特征点不足时，是否降级为单 Homography？
